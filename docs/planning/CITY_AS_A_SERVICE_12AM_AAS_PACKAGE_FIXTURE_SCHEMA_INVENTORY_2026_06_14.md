## Common ladder pieces already proven

The mature family ladders repeatedly use these internal/admin pieces:

```text
fixture_review_gate
→ local_reviewed_fixture
→ internal_package_record
→ operator_read_surface
→ customer_output_schema_gate
→ internal_sample_output
→ sample_output_review_or_hold_decision
```

Some families add stronger pre-approval pieces, such as:

```text
package_review_decision
→ human_operator_approval_request
→ approval_request_read_surface
```

Those stronger pieces are still not approvals. A read surface that displays a future approval request is only a read surface.
