## CRITICAL: Version Sync System INCOMPLETE

**The #1 blocker.** Must be done before ANY more testing.

Currently:
- MCP Server: `/api/v1/version` returns git_sha ‚úÖ
- Dashboard: `/version.json` returns git_sha ‚úÖ
- Lambda Ring 1: `GIT_SHA` env var set at build time, logged at cold start ‚Äî BUT no way to query it externally, and deploys don't verify it
- Lambda Ring 2: same problem

**What's needed:**
1. **Lambda version check in CI**: after `aws lambda update-function-code`, invoke Lambda with a `{"action": "version"}` payload and verify `git_sha` matches `github.sha`. If mismatch, fail the pipeline.
2. **`/api/v1/version/all` endpoint on MCP server**: returns versions of ALL components (self + dashboard + ring1 + ring2). The MCP server can invoke each Lambda with a version check payload.
3. **CI post-deploy step**: calls `/api/v1/version/all` and verifies all components match.
4. **Lambda `:latest` tag problem**: Lambda caches container images. Using `@sha256:digest` instead of `:latest` forces update. The CI must push with a unique tag (`:$GITHUB_SHA`) AND update with that specific tag, not `:latest`.

**File locations:**
- CI: `.github/workflows/deploy.yml` ‚Äî `build-and-push-lambdas` job, `Update Lambda function code` step
- Lambda handlers: `lambda/ring1/handler.py`, `lambda/ring2/handler.py` ‚Äî add version response
- MCP endpoint: `mcp_server/api/routers/misc.py` ‚Äî add `/version/all`
