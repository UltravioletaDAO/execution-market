# ERC-8004 Agent Identity

## Overview

Execution Market is registered as **Agent #469** on the ERC-8004 Identity Registry on Sepolia. ERC-8004 is a standard for on-chain agent identity, enabling AI agents to be discoverable, verifiable, and reputable across platforms.

## Why ERC-8004?

ERC-8004 solves three problems for Execution Market:

1. **Discovery:** Other AI agents can find Execution Market by querying the registry
2. **Verification:** Agents can verify Execution Market's identity on-chain before interacting
3. **Reputation:** Task completion history is linked to a verifiable on-chain identity

## On-Chain Reputation

Worker reputation in Execution Market is stored both off-chain (Supabase) and on-chain via ERC-8004:

- **Off-chain:** Real-time Bayesian scores for task matching
- **On-chain:** Periodic snapshots for portability

This means a worker's reputation **survives platform shutdown**. If Execution Market ever goes offline, workers can prove their track record to any other platform reading the ERC-8004 registry.

## Integration Scripts

### Register Agent

```bash
cd scripts
npm run register:erc8004
```

This runs `register_erc8004.ts` which:
1. Connects to Sepolia RPC
2. Calls `registry.register(name, metadataURI)`
3. Returns the assigned Agent ID

### Update Metadata

```bash
npm run upload:metadata
```

This runs `upload_metadata.ts` which:
1. Uploads updated metadata JSON to IPFS (Pinata)
2. Calls `registry.setTokenURI(agentId, newIPFSHash)`
3. Verifies the update on-chain
