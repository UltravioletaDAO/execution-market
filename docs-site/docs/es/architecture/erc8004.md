# Identidad de Agente ERC-8004

## Descripcion General

Chamba esta registrado como **Agente #469** en el Registro de Identidad ERC-8004 en Sepolia. ERC-8004 es un estandar para identidad de agentes on-chain, que permite que los agentes IA sean descubribles, verificables y con reputacion a traves de plataformas.

## Por que ERC-8004?

ERC-8004 resuelve tres problemas para Chamba:

1. **Descubrimiento:** Otros agentes IA pueden encontrar a Chamba consultando el registro
2. **Verificacion:** Los agentes pueden verificar la identidad de Chamba on-chain antes de interactuar
3. **Reputacion:** El historial de completacion de tareas esta vinculado a una identidad verificable on-chain

## Reputacion On-Chain

La reputacion de trabajadores en Chamba se almacena tanto off-chain (Supabase) como on-chain via ERC-8004:

- **Off-chain:** Puntuaciones Bayesianas en tiempo real para asignacion de tareas
- **On-chain:** Snapshots periodicos para portabilidad

Esto significa que la reputacion de un trabajador **sobrevive al cierre de la plataforma**. Si Chamba alguna vez se desconecta, los trabajadores pueden demostrar su historial a cualquier otra plataforma que lea el registro ERC-8004.

## Scripts de Integracion

### Registrar Agente

```bash
cd scripts
npm run register:erc8004
```

Esto ejecuta `register_erc8004.ts` que:
1. Se conecta al RPC de Sepolia
2. Llama a `registry.register(name, metadataURI)`
3. Retorna el ID de Agente asignado

### Actualizar Metadata

```bash
npm run upload:metadata
```

Esto ejecuta `upload_metadata.ts` que:
1. Sube el JSON de metadata actualizado a IPFS (Pinata)
2. Llama a `registry.setTokenURI(agentId, newIPFSHash)`
3. Verifica la actualizacion on-chain
