# x402r — Contracts Reference

Escrow protocol used by Execution Market for gasless, time-separated authorize/capture payments. This doc is the **contracts-level** cheat sheet. For the full architectural reference (condition system, ABIs, deployment guide, upstream repos), see [[../docs/planning/X402R_REFERENCE.md]].

> **Do not call these contracts directly.** All interactions go through the [[uvd-x402-sdk]] + [[facilitator.ultravioletadao.xyz|Ultravioleta Facilitator]]. Direct calls bypass gas sponsorship and business-logic enforcement. See `CLAUDE.md` → "x402 Payment Architecture".

---

## What is x402r

x402r extends the [x402 payment protocol](https://www.x402.org/) with **escrow capabilities**. Base x402 is one-shot (authorize + settle atomically); x402r introduces a time-separated model where funds can be **locked**, **released**, or **refunded** based on pluggable conditions.

Origin: forked from Coinbase's [commerce-payments](https://github.com/coinbase/commerce-payments). BackTrack added the pluggable condition system, factory-deployed operators, and the gasless facilitator layer.

Key properties:

- **Authorize/Capture split** — funds locked at task creation, released/refunded later.
- **Pluggable access control** — each action (authorize, release, refund) has its own condition contract.
- **Gasless** — Facilitator pays all gas; agents and workers never need native tokens.
- **Safety valve** — payer can reclaim after `authorizationExpiry`; funds cannot be permanently locked.
- **Multichain** — 9 EVM networks with identical interfaces via CREATE2 deterministic addressing. (Solana uses Fase 1 direct SPL transfers — no escrow.)

## Architecture (3 Layers)

```
Layer 3: Facilitator         (off-chain server; pays gas; enforces business logic)
         ────────────────────────────────────────────────────────────
Layer 2: PaymentOperator     (per-config contract; conditions + fee calc)
         ────────────────────────────────────────────────────────────
Layer 1: AuthCaptureEscrow   (shared singleton; holds funds in TokenStore clones)
         ────────────────────────────────────────────────────────────
Layer 0: ERC-20 Tokens       (USDC, EURC, USDT, PYUSD, AUSD)
```

### Layer 1 — `AuthCaptureEscrow`

Shared singleton per chain that actually holds funds. Uses EIP-1167 minimal proxy `TokenStore` clones to isolate funds per payment. All state (authorized amounts, expiries) lives here.

### Layer 2 — `PaymentOperator`

Per-configuration contract deployed via factory. Each operator encodes a specific set of conditions for who may authorize / release / refund, plus a fee calculator. Execution Market deploys one operator per chain, all configured with:

- **Condition:** `StaticAddressCondition` → only the Facilitator EOA can call authorize/release/refund.
- **Fee calculator:** `StaticFeeCalculator(1300 bps)` → 13% protocol fee, split atomically on release (87% worker, 13% operator → flushed to treasury via `distributeFees(USDC)`).

### Layer 3 — Facilitator

Off-chain server operated by Ultravioleta DAO (`https://facilitator.ultravioletadao.xyz`). Holds the whitelisted EOA `0x103040545AC5031A11E8C03dd11324C7333a13C7`. Pays gas for all transactions. Endpoints: `POST /verify`, `POST /settle`, `POST /register`, `POST /feedback`.

## Lifecycle — Execution Market Task (ADR-001)

```mermaid
sequenceDiagram
    participant A as Agent
    participant EM as EM MCP Server
    participant F as Facilitator
    participant E as AuthCaptureEscrow
    participant W as Worker
    participant T as Treasury

    A->>EM: POST /tasks (X-Payment-Auth = EIP-3009 sig)
    EM->>EM: Store pre-auth in DB (no on-chain TX)

    Note over EM: Worker assigned
    EM->>F: /settle (relay agent's auth)
    F->>E: authorize() — locks funds in TokenStore clone
    E-->>F: ok

    Note over W: Worker submits evidence; agent approves
    EM->>F: /settle (release)
    F->>E: release() via PaymentOperator
    E->>W: 87% bounty (net)
    E->>T: 13% fee (via distributeFees)
```

**Cancel paths**

- `lock_on_assignment` (default) + task still `published` → no-op. Pre-auth never used, nothing to refund.
- After escrow lock → `refund()` returns full bounty to agent.
- Pre-auth silently expires at `task.deadline + 1 hour` if no worker is assigned. Zero cost.

**Protocol fee handling.** BackTrack may enable a protocol fee via `ProtocolFeeConfig` (≤5% hard cap, 7-day timelock). When enabled: agent still pays 13% total; x402r deducts its percentage first; worker still receives 100% of bounty; treasury receives `13% − protocol_fee%`. Fully automatic, no EM code changes required.

## Deployment & operations

- **Deploy / redeploy a `PaymentOperator`:** use the `deploy-operator` skill (`.claude/skills/deploy-operator/SKILL.md`). It wraps `scripts/deploy-payment-operator.ts`.
- **Add a new chain or stablecoin:** use the `add-network` skill (`.claude/skills/add-network/SKILL.md`). Update only `sdk_client.py` — everything else auto-derives.
- **Verify multichain health:** run `python scripts/e2e_golden_flow.py` (Golden Flow E2E).
