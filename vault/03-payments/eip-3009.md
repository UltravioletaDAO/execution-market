# EIP-3009

**EIP-3009** is the Ethereum standard for **gasless token transfers** using off-chain signatures. It allows a token holder to authorize a transfer without submitting an on-chain transaction themselves.

## Core Function

```solidity
transferWithAuthorization(
    address from,
    address to,
    uint256 value,
    uint256 validAfter,
    uint256 validBefore,
    bytes32 nonce,
    uint8 v, bytes32 r, bytes32 s
)
```

The signer authorizes, and **any third party** (the [[facilitator]] in our case) can submit the transaction and pay gas.

## Flow in Execution Market

```
1. Agent signs EIP-3009 authorization (off-chain, no gas)
2. SDK sends signed auth to Facilitator
3. Facilitator submits transferWithAuthorization on-chain (pays gas)
4. Tokens move from agent -> recipient (worker, treasury, or escrow)
```

## Why EIP-3009?

- **Gasless for users** — agents and workers never need native tokens (ETH, MATIC, etc.)
- **Atomic** — the entire transfer happens in one transaction
- **Nonce-based** — replay protection built into the standard
- **USDC native** — Circle's USDC implements EIP-3009 on all major chains

## Timestamps

EIP-3009 uses `validAfter` and `validBefore` in **seconds** (not milliseconds). The [[x402-sdk]] handles this automatically.

## Supported Tokens

Only tokens implementing EIP-3009 can use this flow. In practice, this means [[usdc-stablecoins]] on supported networks.
